The Insight

The best dictionary for frame N... is frame N-1.

  RFC 9842 (Web):     HTML page N-1  →  dictionary for page N  →  468× compression
  Carlos Delta (Sim): Frame N-1      →  dictionary for Frame N →   50× compression

  Same math. Different domain.

Between simulation ticks, 98.7% of cells don't change.

XOR the frames. The delta is almost all zeros.

Morton Z-Curve

Interleave bits of (x, y) → spatial neighbors are close in memory.

  • Cache line = 64 bytes
  • One fetch brings 6 hex neighbors
  • Row-major: neighbors scattered across rows
  • Morton: neighbors in same cache line
  • 2.11× speedup at 1 billion cells

L1: <1ns   L2: ~5ns   Main: ~40ns

Why Not zlib?

zlib (CPU)

  • Compress ALL N bytes (95% zeros)
  • LZ77 sliding window on incompressible entropy
  • Lions eating zebras = random bytes
  • zlib burns CPU on chaos
  • 100M cells: 1000ms decode
  • 1B cells: 4600ms decode

Sparse Scatter (GPU)

  • Store ONLY non-zero entries
  • Index + value pairs
  • Zeros are forgotten
  • GPU scatter kernel: 3 lines of Metal
  • 100M cells: 1ms decode
  • 1B cells: 5ms decode

1000× faster. Same file. Same data. Different philosophy.

The Pipeline

  RECORD:
  Metal GPU tick → entity buffer (VRAM, Morton order)
               → XOR with previous frame
               → extract non-zero entries → sparse (index, value) pairs
               → write to NVMe (.savanna file)

  PLAY:
  NVMe → read sparse pairs into unified memory
       → GPU kernel: scatter XOR into frame buffer      ← 3 lines of Metal
       → GPU kernel: de-Morton to row-major              ← 1 pass
       → GPU kernel: LOD downsample to 2048²             ← majority vote
       → HTTP → WebGL browser

  CPU touches ZERO cell data during playback.

Unified memory on Apple Silicon: no CPU↔GPU copy. Same bytes.

I-Frame / P-Frame Architecture

  Frame:  0    1    2   ...  59   60   61  ...  119  120
  Type:   I    P    P   ...  P    I    P   ...   P    I
          ↑                       ↑                   ↑
       keyframe              keyframe             keyframe

  I-frame: full frame (zlib, rare — 1 per 60)
  P-frame: sparse scatter (index, value) pairs only
  • Random access: seek nearest I-frame, replay ≤60 P-frames
  • O(60) not O(N) — the dumbbell chain
  • Each rigid link is self-contained
  • Inspired by 3B-Backup-IP coprime basket rotation

.savanna Format (SDLT v3)

  Header (32 bytes):
    "SDLT" (4B)  width (u32)  height (u32)  n_frames (u32)
    total_cells (u64)  keyframe_interval (u32)  format (u32)

  I-Frame:  compressed_size (u32) + zlib(full_frame)

  P-Frame (sparse):
    total_size (u32)
    count (u32)                    — number of changed cells
    indices [u32 × count]          — Morton indices
    values  [u8  × count]          — XOR values

Backward compatible: v1 (24B header), v2 (28B), v3 (32B). Auto-detected.

From Web to Science

Your RFC 9842 Dictionary TTL enables this:

  • Web: Previous HTTP response = compression dictionary
  • Simulation: Previous frame = compression dictionary
  • Genome: Previous chromosome = compression dictionary
  • Weather: Previous time step = compression dictionary
  • Video: Previous frame = compression dictionary

The principle is universal. The previous state is always the best predictor.
The encoding is always the diff. The innovation is making it GPU-native.
