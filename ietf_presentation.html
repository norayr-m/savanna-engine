The Problem

Without Dictionary Persistence,

RFC 9842 Dictionary TTL solves this: persist the previous response as the compression dictionary for the next one.

Decode Performance

Hardware-Accelerated Dictionary Decode

Standard Decode (CPU zlib)

  • Decompress full dictionary delta on CPU
  • LZ77 sliding window
  • Sequential, single-threaded
  • 100M cells: 1,000 ms
  • 1B cells: 4,600 ms
vs

Sparse Scatter (GPU)

  • Store only non-zero delta entries
  • GPU kernel: scatter XOR in parallel
  • Massively parallel, hardware-native
  • 100M cells: 1 ms
  • 1B cells: 5 ms

1,000× faster decode when the dictionary delta format is designed for the receiver's hardware.

Dictionary Delta Format

Sparse Scatter Encoding

// Dictionary delta: only store what changed // 98.7% of entries are zero → not stored P-Frame (dictionary delta): count: uint32 // number of changed entries indices: [uint32 × count] // positions that changed values: [uint8 × count] // XOR delta values I-Frame (full dictionary): zlib_compressed(full_state) // every 60th frame // GPU decode kernel — the entire decoder: kernel void scatter(frame[indices[gid]] ^= values[gid]);

The entire decode is one GPU kernel, one line of logic. The format is the decoder.

Four Key Arguments

1. Empirical Evidence

50× compression measured on real scientific data. Not theoretical. Not synthetic benchmarks. A living ecosystem simulation at billion-cell scale.

2. Scale Invariance

The compression ratio holds from 1M to 100B cells. The principle is independent of payload size. Web payloads sit in the same redundancy range.

3. Hardware Acceleration

Dictionary deltas can be designed for GPU decode. 1,000× faster than zlib. Format-as-decoder: the encoding IS the decode instruction.

4. The Web Case is a Subset

If dictionary persistence helps at 98.7% redundancy on spatial tensors, it helps at 80-99% redundancy on JSON APIs, HTML pages, and real-time streams.

Full Pipeline

Dictionary Transport Architecture

// Sender (simulation or web server) current_state = compute(previous_state) delta = current_state XOR previous_state // dictionary delta sparse_entries = extract_nonzero(delta) // 1.3% of data send(sparse_entries) // 50× smaller // Receiver (client or browser) sparse_entries = receive() gpu_scatter(frame_buffer, sparse_entries) // microseconds render(frame_buffer) // immediate // The dictionary IS the previous state. // RFC 9842 TTL keeps it alive between requests. // Without TTL: dictionary evicted, send full state. 50× waste.
